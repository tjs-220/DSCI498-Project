# DSCI498-Project

This project investigates how diffusion‑based generative models can be adapted to perform effectively in low‑resource environments, focusing on scenarios where only a small number of training samples are available. Because diffusion models typically rely on large and diverse datasets, their behavior under extreme data scarcity is an important practical question for domains such as handwritten scripts, medical imaging, and other specialized visual categories. Using the Omniglot dataset—an ideal benchmark due to its naturally limited number of samples per character—the project evaluates lightweight strategies for few‑shot adaptation, including augmentation‑driven regularization, low‑rank fine‑tuning, and adjustments to classifier‑free guidance. The implementation provides a reproducible experimental framework built around a compact UNet architecture, a DDPM‑style diffusion schedule, and a few‑shot data loader that constructs small class‑balanced subsets from Omniglot. To run the code, users should install Python 3.9 or later and set up a virtual environment before installing required packages such as PyTorch, Torchvision, TensorFlow or TensorFlow‑CPU, TensorFlow Datasets, NumPy, Pillow, and TQDM. After installing dependencies, training can be launched by running python main.py, which loads Omniglot through TFDS, constructs a few‑shot subset, trains the diffusion model, saves checkpoints, and generates sample images. Sampling can also be performed independently by calling the sampling script with a checkpoint path. The codebase is organized into modular components for configuration, training, sampling, dataset handling, the diffusion schedule, and the UNet model, enabling clear experimentation and modification. Overall, the project provides an accessible, open‑source implementation and research‑style analysis that together clarify the feasibility, limitations, and practical considerations of applying diffusion models to severely constrained data settings.
